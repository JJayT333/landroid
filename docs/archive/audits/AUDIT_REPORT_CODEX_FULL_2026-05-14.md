## 1. Executive verdict

Verdict: conditional no-go for inviting additional hosted/cloud testers until two hosted-boundary issues are fixed. The repo is strong on deterministic math, local persistence, test volume, and AWS proxy guardrails, but the browser Cognito configuration appears to point OIDC issuer/JWKS at the Hosted UI domain even though the working Cognito metadata lives at the user-pool issuer URL, and hosted "read-only" AI still exposes a focus-changing tool that writes through autosave (`src/auth/AuthProvider.tsx:41`, `src/auth/AuthProvider.tsx:48`, `src/auth/AuthProvider.tsx:49`, `customHttp.yml:15`, `src/ai/runChat.ts:57`, `src/ai/tools.ts:835`, `src/ai/tools.ts:899`, `src/main.tsx:85`).
