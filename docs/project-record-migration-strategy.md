# Project Record Migration Strategy

Status: Phase 1 schema foundation. No `.landroid` file-format change ships in
this phase.

## Current Format

The live package format remains `LANDROID_FILE_VERSION = 8`. Export/import still
serializes the current `WorkspaceData` snapshot plus side-store sections for
documents, owners, maps, research, curative records, and canvas state. Phase 1
adds project-record schemas and pure projection/adapters beside that path; it
does not write records into `.landroid` files and does not change runtime store
hydration.

## Future Write Strategy

The first `.landroid` format that includes project records must be an explicit
new version, not an unannounced extension of v8. The export writer should:

- keep the complete local snapshot and side-store payload as the rollback
  source
- add a validated project-record bundle generated by
  `buildProjectRecordsFromWorkspace`
- validate the bundle with `ProjectRecordBundleSchema` before writing the file
- include document bytes only in the existing package sections; record rows
  should reference blob-backed evidence by content hash and vault object ID
- keep `workspaceId`, `projectId`, `lastModified`, `revision`, tombstone, and
  sync-state metadata on record envelopes for future sync

## Future Import Strategy

Import must continue to dispatch by file version. The safe import sequence for a
record-bearing version is:

1. Parse the package envelope and reject unknown future versions before any
   store replacement.
2. Validate the existing snapshot/side-store sections through the current v8
   normalizers.
3. Validate the project-record bundle with `ProjectRecordBundleSchema`.
4. Rebuild or compare projections such as `MathInputView` against the snapshot
   before any cutover claim.
5. Replace side stores with the existing rollback-safe path.
6. Keep the snapshot authoritative until a later phase proves workflow parity
   and explicitly migrates a surface to records.

## Required Tests Before Format Change

- v8 import compatibility and future-version rejection
- `.landroid` export/import round trip with the record bundle present
- rollback-safe side-store replacement when record validation fails
- serialization round trip for every declared record type
- `MathInputView` parity against Phase 0 goldens for decimal/fraction display,
  lease allocation order, warning-only states, and jurisdiction isolation
